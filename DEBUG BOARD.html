COMO USAR
DEB BOARD

la aplicacion deb board se puede usar para realizar las pruebas por etapas o para debugear componentes

para abrir la aplicacion es necesario parar la prueba con la obcion de stop de la pantalla de basic

escribir el comando deb board,dar enter y esperar un momento

despues de un momento aparecera la pantalla principal

para realizar las pruebas por etapas es necesario ir a la opcion testplan macros

una vez seleccionado la opcion testplan macros,se despliega las opciones de pruebas por etapas y se deben de comenzar a realizar de arriba hacia abajo, es mas comun que los componentes a debugear fallen en la etapa de analogos

los componentes que fallen se mostraran en una lista a lado izquierdo

para comenzar a debugear un componente procede a selecionar de la lista seccionando las siguientes opciones
debug
debug selected test

posteriormente se mostrara una ventana con la informacion del componente
proceder a buscar la sintaxis de configuracion

la sintaxis de configuracion del componente muestra varios valores
proceder a validar modicando el valor de re

la sintaxis de configuracion del componente muestra varios valores
al proceder a modicar este valor cambias el valor de low limit

la sintaxis de configuracion del componente muestra varios valores
al proceder a modicar este valor cambias el valor de hight limit

cada que se modifique un valor debemos correr la prueba del componente en compile and go para validar

si la unida falla proceder a modificar el valor que se debugeo

si la unidad paso se procede a compilar y guardar los cambios

para compilar los cambios se deben de seguir los siguientes pasos
seleccionar la opcion debug
seleccionar la opcion compile test
aparecera una ventana
seleccionar yes
aparecera una ventana
seleccionar ok
validar la leyenda object produced en la ventana de basic

una vez compilado se procede a guardar los cambios
seleccionar opcion file
seleccionar opcion save

para debugear otro componente de la lista
seleccionar opcion debug
seleccionar opcion debug board
